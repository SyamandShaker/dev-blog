The FHS (Filesystem Hierarchy Standard)

The "standard" directory structure is described by the FHS (Filesystem Hierarchy Standard) to which most Linux distributions try to conform.

What is the FHS?

The FHS (Filesystem Hierarchy Standard) is a specification that defines the file system hierarchy of a Linux operating system. It specifies the minimum requirements for high-level subdirectories, such as /usr and /bin, and which files and symbolic links must be located in these directories. In an FHS-compliant operating system, all files and directories are anchored in a top-level directory named / (a single slash).

The main directories are:

/bin :

contains the basic commands (under Unix, almost all commands are "external", and not integrated in the shell like under DOS).

/boot :

contains the information necessary to start the machine.

/dev :

contains the special files corresponding to the devices.

/etc :

contains most of the configuration files.
