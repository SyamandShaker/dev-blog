Git-SCM

Git is a distributed version control system that allows you to track changes in source code during software development. It is designed to coordinate the work of programmers, but it can be used to track changes in any set of files. Its goals are speed, data integrity, and support for distributed and non-linear workflows.
Git is easy to learn and has a small footprint with lightning-fast performance. It outperforms content management tools such as Subversion, CVS, Perforce and ClearCase with features such as inexpensive local branches, convenient transit areas and multiple workflows.
SCM {software configuration management } is a practice for tracking and controlling changes to development software. It includes all the processes involved in building, packaging, and deploying software.
